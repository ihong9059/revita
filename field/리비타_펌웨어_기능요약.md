# 리비타 펌웨어 기능 요약

> 원본: 26-03-01 펌웨어 개발 의뢰서

---

## 1. 제품 개요

### 리비타 타워
- **역할**: 노지 스마트팜 관제 장치 (중앙 허브)
- **주요 기능**:
  - LTE로 서버와 통신
  - LoRa로 리비타 링크들과 통신 (마스터/게이트웨이)
  - PTZ 카메라로 현장 촬영
  - RS485 센서로 조도 데이터 수집

### 리비타 링크
- **역할**: 노지 스마트팜 단말기 (센서/제어 노드)
- **주요 기능**:
  - LoRa로 리비타 타워와 통신
  - RS485 센서 데이터 수집
  - DIO로 밸브/모터 제어
  - 유량계 측정

## 2. 핵심 요구 성능

### 2.1 동적 설정 기능

| 기능 | 설명 |
|------|------|
| **통합 config 파일** | 한 기기의 모든 설정을 한 파일로 관리, 재부팅 후에도 유지 |
| **동적 스케줄 관리** | config로 작업 주기, 시작 시간, 활성화 여부를 실시간 수정 |
| **가상 장비 매핑** | 센서 드라이버가 config에 내장, 드라이버 교체만으로 다양한 장치 지원 |
| **플러그 앤 플레이** | 장비 연결 시 "기기 추가" 명령으로 드라이버 활성화 |

### 2.2 저전력 대기

| 기능 | 설명 |
|------|------|
| **스케줄 기반 wake** | 지정된 시간에만 깨어나서 동작 수행 |
| **인터럽트 기반 wake** | 통신 수신, 버튼, 진동 센서 등에 의해 깨어남 |
| **외부 전원 제어** | 외부 기기 사용할 때만 전원 공급 (MOSFET 스위칭) |
| **상시 sleep** | IDLE 시간에는 deep/light sleep 유지 |

### 2.3 원격 가용성 및 유지보수

| 기능 | 설명 |
|------|------|
| **자체 복구** | 오작동 발생 시 특정 시간 내 리셋으로 정상 복구 |
| **Dual-bank OTA** | LoRa, BLE, LTE로 펌웨어 업데이트, 실패 시 롤백 |
| **원격 로그 분석** | 상세 로그로 현장 방문 없이 문제 파악 |
| **물리적 강제 복구** | 버튼 특정 패턴으로 공장 초기화 |

### 2.4 통신 및 보안

| 기능 | 설명 |
|------|------|
| **통신 보안** | LTE, LoRa 인증서 기반 보안 |
| **통신 fallback** | 통신 두절 시 자동 재시도, ACK 확인 |
| **명령 fallback** | 중복 실행 방지, 수행 불가 상태면 거부, 취소 기능 |
| **시간 동기화** | 서버와 시간 동기화로 정확한 스케줄 실행 |

## 3. 드라이버 구성

### 3.1 공통 드라이버

| 드라이버 | 기능 |
|---------|------|
| **LoRa** | 타워-링크 간 통신, MeshCore 라이브러리 사용 |
| **BLE** | 사용자 앱 연결, OTA, 설정 |
| **Flash** | OTA용 dual-bank 저장소 |
| **RS485** | 센서 데이터 수집 (MODBUS) |
| **전원 버튼** | 상태 확인, BLE 페어링, 기기 활성화 |
| **도난 방지** | 진동 감지 → 부저 알림 → 통신 전송 |

### 3.2 리비타 링크 전용

| 드라이버 | 기능 |
|---------|------|
| **3-Line 모터 제어** | 밸브 개방/폐쇄 (CR02, CR03 타입) |
| **유량계** | 펄스 카운팅으로 유량 측정 (sleep 중 background counting) |

### 3.3 리비타 타워 전용

| 드라이버 | 기능 |
|---------|------|
| **LTE + GPS** | 서버 통신 (MQTT), 위치 정보 획득 |
| **프로세서 제어** | Core3506에 제어권 양도하여 카메라 촬영 |

## 4. 스케줄 시스템

### 4.1 스케줄 종류

| 종류 | 설명 | 예시 |
|------|------|------|
| **Task** | 특정 시점에 1회 실행 | 2026/03/06 10:00에 관수 시작 |
| **Cron** | 반복 실행 (매일/매분) | 매일 10:00에 촬영, 10분마다 센서 리딩 |

### 4.2 Task 상태 흐름

```
PENDING → RUNNING → COMPLETED → REPORTED → 삭제
                 ↘ FAILED ↗
                 ↘ CANCELED
                 ↘ EXPIRED (방전/시간 초과)
```

### 4.3 주요 Task/Cron 목록

| 이름 | 타입 | 기능 |
|------|------|------|
| **관수 밸브 제어** | Task | 밸브 개방/폐쇄, 자동 타임아웃 |
| **유량계 복합 제어** | Task | 목표 유량 도달 시 밸브 닫기 |
| **RS485 센서 리딩** | Cron | N분마다 센서 데이터 수집 및 전송 |
| **기기 상태 확인** | Cron | 배터리, 온도, 로그 등 상태 체크 |
| **프로세서 wakeup** | Task/Cron | 카메라 촬영을 위한 프로세서 구동 |
| **부저 알림** | Task | 기기 찾기용 1분간 부저 |
| **LoRa 송신** | Task | 다른 링크 동작을 위한 메시지 전송 |

## 6. 버튼/LED 인터페이스

### 버튼 입력 패턴

| 패턴 | 동작 |
|------|------|
| 짧게 1번 | 현재 상태를 LED로 5초간 표시 |
| 3초 길게 | BLE Advertising 시작 |
| 10초 길게 | 기기 활성화 (비활성화→활성화) |

### LED 상태 표시

| 패턴 | 의미 |
|------|------|
| 1박자 점멸 (딱, 딱) | 활성화 상태 - 정상 |
| 2박자 점멸 (따딱, 따딱) | 활성화 상태 - 저전력 |
| 항상 켜짐 | BLE 페어링 중 |
| 3박자 점멸 | 기기 활성화 완료 |

## 8. 통신 구조

### 리비타 타워 통신

```
서버 ←──LTE/MQTT──→ 리비타 타워 ←──LoRa──→ 리비타 링크들
                         ↑
                    USB/UART
                         ↓
                   프로세서(Core3506) ←──ETH──→ PTZ 카메라
```

### 리비타 링크 통신

```
리비타 타워 ←──LoRa──→ 리비타 링크 ←──RS485──→ 센서
                              ↑
                             GPIO
                              ↓
                         밸브/유량계
```

## 9. 전원 관리

### 리비타 타워
- 12V 배터리 + 18V 태양광 충전
- 5V/3.3V 레귤레이터
- 각 부품별 MOSFET 전원 스위칭

### 리비타 링크
- 1S 인산철 배터리 + 태양광 충전
- 3.0V watchdog (저전압 시 RST)
- 12V step-up (부저, 밸브용)

## 10. 안전 장치

| 기능 | 설명 |
|------|------|
| **3.0V watchdog** | 배터리 3.0V 이하 시 자동 리셋 |
| **밸브 fallback** | 재부팅 시 밸브 상태 확인 → 자동 폐쇄 |
| **OTA rollback** | 펌웨어 업데이트 실패 시 이전 버전 복구 |
| **config 무결성** | 작성 중 리셋 발생해도 복구 가능 |
| **명령 타임아웃** | 밸브 개방 후 지정 시간 내 폐쇄 명령 없으면 자동 폐쇄 |
