**설치 위치**: `/home/uttec/revita/MeshCore`

#### 2.3 ARM 툴체인 연동

기존 설치된 ARM 툴체인(`~/gnuarmemb`)을 PlatformIO에서 사용하도록 설정:

1. `~/gnuarmemb/package.json` 생성
2. `MeshCore/platformio.ini`에 툴체인 경로 추가

```ini
[nrf52_base]
platform_packages =
  framework-arduinoadafruitnrf52 @ 1.10700.0
  toolchain-gccarmnoneeabi@symlink:///home/uttec/gnuarmemb
```

### 5. RAK4630 vs RAK4631

| 항목 | RAK4630 | RAK4631 |
|------|---------|---------|
| 형태 | 베어 모듈 (SiP) | WisBlock Core 모듈 |
| 칩셋 | nRF52840 + SX1262 | 동일 |
| 호환성 | RAK19007 베이스보드 사용 시 RAK4631 펌웨어 호환 |

MeshCore에는 RAK4631 variant만 제공되므로, RAK4630 + RAK19007 조합에서 RAK4631 펌웨어 사용.

### 10. RAK4630 기술 검토 요약

#### 10.1 하드웨어 사양

| 항목 | 사양 |
|------|------|
| MCU | Nordic nRF52840 (ARM Cortex-M4F) |
| LoRa | Semtech SX1262, 22dBm 최대 출력 |
| 주파수 | KR920 (한국), AS923, EU868 등 |
| Sleep 전류 | 4.23 uA |
| 크기 | 15 x 23 x 3 mm |

#### 10.2 한국 법규 주의사항

```
중요: 한국에서 LoRa 사용 시 출력 제한 필요
- 법적 최대 출력: 14dBm (25mW)
- 현재 MeshCore 설정: 22dBm
- 수정 필요: LORA_TX_POWER=14
```

#### 10.3 예상 통신 거리

| 환경 | 예상 거리 |
|------|-----------|
| 개활지 | 5-10 km |
| 도심 | 1-3 km |
| 실내 | 100-500 m |
