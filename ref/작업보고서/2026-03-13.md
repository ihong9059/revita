#### 3.2 REVITA_LINK Variant 생성

**목적**: REVITA_LINK 보드에 맞게 커스터마이징 (디버그 출력을 Serial1으로 변경)

**생성 파일**: `/home/uttec/revita/meshtastic/variants/nrf52840/revita_link/`

**platformio.ini 주요 설정**:
```ini
[env:revita_link]
extends = nrf52840_base
board = wiscore_rak4631
build_flags = ${nrf52840_base.build_flags}
  -I variants/nrf52840/revita_link
  -D REVITA_LINK
  -D USER_DEBUG_PORT=Serial1
  -DMESHTASTIC_EXCLUDE_MQTT=1
  -DMESHTASTIC_USE_EINK_UI=0
```

**빌드 결과**: 성공
- RAM: 40.9% (101652 bytes)
- Flash: 86.8% (707476 bytes)

#### 3.4 시리얼 출력 문제

**현상**: cutecom에서 /dev/ttyUSB0 (115200 baud) 연결 시 출력 없음

**분석**:
- `USER_DEBUG_PORT=Serial1` 설정했으나 출력 안됨
- SerialConsole.cpp에서 `Port.begin(SERIAL_BAUD)` 호출
- Serial1 핀 설정 (PIN_SERIAL1_TX=16, PIN_SERIAL1_RX=15)은 정상

**가능한 원인**:
1. Serial1 초기화 타이밍 문제
2. Meshtastic 내부 초기화 순서 문제
3. 추가 핀 설정 필요

#### 3.5 MCP23017 핀 맵 수정 (REVITA_LINK_v1.pdf 기준)

**목적**: 기존 문서의 핀 맵이 실제 회로도와 불일치하여 정정

**GPA (Port A) - 출력**:
| 핀 | 신호명 | 방향 | 기능 |
|----|--------|------|------|
| GPA0 | (미사용) | - | - |
| GPA1 | X_EN_A | OUT | X축 모터 A방향 |
| GPA2 | X_EN_B | OUT | X축 모터 B방향 |
| GPA3 | X_EN_P2 | OUT | X축 모터 전원 (GND 스위치) |
| GPA4 | Y_EN_A | OUT | Y축 모터 A방향 |
| GPA5 | Y_EN_B | OUT | Y축 모터 B방향 |
| GPA6 | Y_EN_P2 | OUT | Y축 모터 전원 (GND 스위치) |
| **GPA7** | **12V_EN** | **OUT** | **12V 부스트 활성화** |

**GPB (Port B) - 혼합**:
| 핀 | 신호명 | 방향 | 기능 |
|----|--------|------|------|
| **GPB0** | **BUZZER_EN** | **OUT** | **부저 활성화** |
| GPB1 | (미사용) | - | - |
| GPB2 | RE# | OUT | RS485 수신 enable (Active Low) |
| GPB3 | RST# | OUT | 외부 리셋 |
| GPB4 | DE | OUT | RS485 송신 enable |
| GPB5 | LED_EN | OUT | LED enable |
| GPB6 | BTN | IN | 외부 버튼 입력 |
| GPB7 | VIB_SENSE | IN | 진동 센서 입력 |

**nRF52840 직접 연결 (MCP23017 아님)**:
| 핀 | 신호명 | 방향 | 기능 |
|----|--------|------|------|
| P0.10 | DIO_X | IN | 12V 디지털 입력 X |
| P0.09 | DIO_Y | IN | 12V 디지털 입력 Y |
