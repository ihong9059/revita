### 3. 시도한 접근 방법

#### 3.1 Arduino Framework (실패)

**문제점**:
- Adafruit nRF52 Arduino 프레임워크는 Softdevice S140 + Bootloader 환경을 전제로 동작
- J-Link로 직접 flash 시 초기화 코드 실행 실패로 HardFault 발생
- `Serial1.begin()` 호출 시 내부적으로 핀 매핑 불일치 문제

**시도한 설정**:
```ini
[hw_test_base]
platform = nordicnrf52
board = rak4631
framework = arduino
build_flags =
  -I variants/rak4631
  -D RAK_4631
lib_deps =
  adafruit/Adafruit TinyUSB Library
```

**결과**: HardFault - Serial 출력 없음

#### 3.2 Zephyr flash_test (성공)

**목적**: 하드웨어 자체 문제인지 확인

```bash
JLinkExe -SelectEmuBySN 001050234191 -CommandFile /tmp/jlink_zephyr_flash.txt
```

**결과**:
- 카운터 출력 정상 확인 (0, 1, 2, ...)
- 하드웨어 정상 동작 확인
- UART2 핀 (P0.16, P0.15) 정상

#### 3.3 Bare-metal UART (성공)

**접근 방법**: Arduino/Softdevice 없이 nRF52840 레지스터 직접 제어

#### 4.3 linker.ld

```ld
MEMORY
{
    FLASH (rx)  : ORIGIN = 0x00000000, LENGTH = 1M
    RAM (rwx)   : ORIGIN = 0x20000000, LENGTH = 256K
}

SECTIONS
{
    .text :
    {
        KEEP(*(.vectors))
        *(.text*)
        *(.rodata*)
    } > FLASH

    .data :
    {
        *(.data*)
    } > RAM AT > FLASH

    .bss :
    {
        *(.bss*)
        *(COMMON)
    } > RAM
}
```

#### 4.4 main.c 핵심 코드

```c
// nRF52840 UARTE0 레지스터 직접 접근
#define NRF_UARTE0_BASE     0x40002000
#define NRF_P0_BASE         0x50000000

#define TX_PIN 16
#define RX_PIN 15
#define BAUDRATE_115200 0x01D7E000

void uart_init(void) {
    // TX 핀 출력 설정
    GPIO_PIN_CNF(TX_PIN) = 0x00000001;
    GPIO_PIN_CNF(RX_PIN) = 0x00000000;

    // UART 설정
    UARTE0_ENABLE = 0;
    UARTE0_PSEL_TXD = TX_PIN;
    UARTE0_PSEL_RXD = RX_PIN;
    UARTE0_PSEL_RTS = 0xFFFFFFFF;
    UARTE0_PSEL_CTS = 0xFFFFFFFF;
    UARTE0_BAUDRATE = BAUDRATE_115200;
    UARTE0_CONFIG = 0;
    UARTE0_ENABLE = 8;
}
```

### 12. 중요 발견: GPIO PIN_CNF 설정 (반복 실수 방지!)

**문제**: Bare-metal UART 데이터 전송 후 시리얼 출력 안됨

**원인**: GPIO PIN_CNF 레지스터 설정 오류

```c
// 잘못된 설정 (작동 안함!)
P0_PIN_CNF(TX_PIN) = 0x00000001;  // DIR=Output만 설정
P0_PIN_CNF(RX_PIN) = 0x00000000;  // 기본값

// 올바른 설정 (작동함!)
P0_PIN_CNF(TX_PIN) = 0x00000003;  // DIR=Output, INPUT=Disconnect
P0_PIN_CNF(RX_PIN) = 0x0000000C;  // DIR=Input, PULL=Pullup
```

**검증 방법**:
1. `~/revita/hwTest/serial_counter/` Zephyr 프로그램으로 하드웨어 먼저 확인
2. Zephyr 작동 → Bare-metal 코드 문제 확정
3. GPIO 설정 수정 후 해결
