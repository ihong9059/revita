### 3. 주요 작업 내용

#### 3.1 하드웨어 구성 확인

- **타겟 보드**: RAK4631 (RAK4630 모듈, nRF52840)
- **플래시 방법**: PCA10056 J-Link (S/N: 1050295470) 경유, JLinkExe 스크립트
- **디버그 콘솔**: uart1 TX=P0.16 → CP210x → /dev/ttyUSB0, 115200bps
- **uart1 RX 제거**: P0.15를 LED_EN으로 사용, uart1 RX는 P0.02(dummy)로 이동

#### 3.2 Phase 1 개별 드라이버 테스트 (6개 완료)

각 드라이버별 독립 프로젝트로 검증:

| 프로젝트 | 테스트 내용 | 핀 |
|---------|-----------|-----|
| hwTest_power_buzzer | 12V/부저/X/Y모터 | P0.17, P0.24, P0.14/13, P0.25/P1.01 |
| hwTest_digital_input | BTN/VIB 입력 | P0.05(Pull-down), P0.21(Pull-up) |
| hwTest_rs485 | RS485 송수신 | TX=P0.20, RX=P0.19, DE=P1.04, RE#=P1.03 |
| hwTest_rs485_led | RS485 + LED 점멸 | 위 + LED_EN=P0.15 |
| hwTest_adc | 배터리 전압 ADC | P0.31(AIN7), VCC/2 분압 |
| hwTest_flash | QSPI Flash R/W | MX25R1635F, erase/write/read verify |

#### 3.3 Phase 2: Modbus RTU 통합 펌웨어 (kc_firmware)

모듈 구조로 구현:

```
kc_firmware/src/
├── main.c        # 메인 루프, 초기화
├── modbus.c/.h   # Modbus RTU 프레임 처리, CRC-16
├── rs485.c/.h    # UART0 송수신, DE/RE# 반이중 제어
├── registers.c/.h # 레지스터 맵, 읽기/쓰기 핸들러, 센서 업데이트
└── drivers.c/.h  # GPIO/ADC/Flash 드라이버 래퍼
```

구현된 Modbus 기능:
- Function 0x03 (Read Holding Registers)
- Function 0x06 (Write Single Register)
- Function 0x10 (Write Multiple Registers)
- 예외 응답 (Illegal Function/Address/Value)
- CRC-16 검증

구현된 레지스터:
- 0x0000~0x0004: 기기 정보 (ID, 버전, 시험모드)
- 0x0010~0x0016: 전원/센서 (배터리, 진동, 버튼, 12V, 입력변화플래그)
- 0x0020~0x0023: 밸브 상태
- 0x0040~0x0041: 부저/LED 상태
- 0x0070~0x0071: Flash 상태
- 0x0100~0x0103: 밸브 제어 (자동 정지 타이머 포함)
- 0x0120~0x0122: 부저/LED/12V 제어
- 0x0150: Flash 셀프테스트
- 0x01F0: 비상 정지 (ALL STOP)
- 0x01FF: 시험 모드 진입/해제

입력 변화 감지 기능 (0x0016):
- BTN/VIB 상태 변화를 플래그로 기억
- Host 폴링 시 변화 여부 전달 → 읽힌 후 자동 클리어

device nRF52840_xxAA
si swd
speed 4000
connect
h
loadfile [hex파일경로]
r
g
exit
EOF
JLinkExe -CommanderScript /tmp/flash_hwtest.jlink

# Host Web UI 실행
cd /home/uttec/revita/zephyr_workspace/kc_cert/rs485
python3 host/app.py /dev/ttyUSB1 9600 5000
# 브라우저: http://192.168.0.29:5000
