### 3. 주요 작업 내용

#### 3.1 RAK4630 교체 및 J-Link 확인

- Link 보드의 RAK4630 모듈 교체 (2회)
- 첫 번째 모듈: `Failed to attach to CPU` → SWD 접촉 불량
- 두 번째 모듈: 정상 연결, adc_test/node 플래시로 검증
- mass erase (`--erase`) 옵션으로 새 모듈 초기화

#### 3.2 LED 테스트 (led_test/node, led_test/tower)

- **Node (Link)**: P0.15 (UART2_RX 겸용) → GPIO 출력으로 사용
  - UART1 RX를 P0.19로 이동하여 P0.15를 해제
  - 1초 간격 ON/OFF 토글 확인
- **Tower**: MCP23017 GPB6 (LED_EN)
  - I2C0 핀 명시 (P0.13 SDA, P0.14 SCL, 100kHz)
  - MCP23017 init FAIL (rc=-2) — I2C scan에서 디바이스 미발견
  - **이전 세션(4/7)의 buzzer_test/tower에서는 동작했음** → HW 상태 변화 의심

#### 3.3 Digital Input 테스트 (input_test/node, input_test/tower)

- **Node**: BTN=P0.05 (pull-up), VIB_SENSE=P0.21 (pull-down)
- **Tower**: BTN=P1.02 (pull-up), VIB_SENSE=P1.04 (pull-down)
- 100ms 폴링, 상태 변화 시 즉시 출력, 1초마다 상태 표시

#### 3.4 Motor Driver 테스트 (motor_test/node_x, motor_test/node_y)

- **X 채널**: EN_A=P0.14, EN_B=P0.13, EN_P2=P0.04
- **Y 채널**: EN_A=P0.25, EN_B=P1.01, EN_P2=P1.02
- **중요**: EN_P2는 반드시 LOW 유지 (motor 동작과 무관)
- 패턴: 3s CW → 1s STOP → 3s CCW → 1s STOP × 3회 → 정지
- CW: EN_A=HIGH, EN_B=LOW / CCW: EN_A=LOW, EN_B=HIGH / STOP: all LOW
- 12V_EN=P0.17 HIGH 필요

#### 3.5 RS485 Modbus RTU 센서 테스트 (soil_sensor_test/node)

- Modbus RTU 마스터 구현: CRC16-Modbus (poly 0xA001, init 0xFFFF)
- **RS485 loopback 진단**: DE+RE# 동시 활성으로 TX→RX 경로 확인
- **Slave ID 스캔**: 0x01~0x10 순회, 응답 있는 ID 자동 감지
- 결과:

| 센서 | Slave ID | Baud rate | 습도 | 온도 |
|------|----------|-----------|------|------|
| 토양센서 #1 | 0x01 | 4800 | 0.0% | 22.9°C |
| 토양센서 #2 | 0x01 | 9600 | 0.0% | 22.6°C |
| 엽온센서 | 0x02 | 9600 | 0.0% | 22.4°C |

- **핵심 발견**: 토양센서 baud rate가 제품마다 다름 (4800 또는 9600)

#### 3.6 USB CDC 테스트 (usb_test/tower 수정)

- 기존: console이 USB CDC로만 출력 → ttyUSB1 출력 없음
- 수정: console을 uart0(ttyUSB1)으로 복원 + USB CDC에 수동 출력
- `[UART]` / `[USB]` 접두어로 출력 경로 구분
- ttyACM4 = REVITA TOWER USB CDC (VID 0x2FE3, PID 0x0100)

#### 3.7 Host 시뮬레이션 웹 (host_test/web + host_test/tower)

- **Tower 펌웨어**: USB CDC로 텍스트 라인 수신 → 명령 파싱 → debug UART 출력 + 응답
  - 명령: CMD:LED_ON, CMD:LED_OFF, CMD:BUZZER_ON/OFF, CMD:SENSOR_READ, CMD:STATUS, DATA:xxx
  - 응답: ACK:xxx, SENSOR:temp=22.5,humi=55.0, STATUS:uptime=N,rx_count=N
- **Flask 웹 UI**: 버튼으로 명령 전송, 통신 로그 실시간 표시
  - 포트 8080에서 서비스 (http://192.168.0.29:8080)
  - pyserial로 /dev/ttyACM4 연결

### 7. 핵심 파악 사항

1. **토양센서 baud rate가 제품마다 다름**: 같은 모델이라도 4800 또는 9600 — slave ID 스캔 + 다중 baud rate 시도 필요

2. **Motor EN_P2는 반드시 LOW**: motor CW/CCW 동작과 무관, HIGH로 하면 안 됨

3. **Tower MCP23017 여전히 불안정**: 이전 세션(4/7) buzzer_test에서는 동작했으나 오늘 led_test에서 rc=-2 — HW 접촉 또는 모듈 상태 변화 의심

4. **USB CDC ttyACM 번호**: J-Link가 ttyACM0~3 차지, Tower USB CDC는 ttyACM4 — `udevadm info`로 ID_MODEL 확인 필요

5. **Flask 웹 포트 5000 차단**: 다른 PC에서 접속 불가 → 포트 8080으로 변경 시 정상 동작
