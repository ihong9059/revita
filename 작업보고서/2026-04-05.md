### 3. 주요 작업 내용

#### 3.1 system/lte Zephyr 프로젝트 생성

system/tower 프로젝트를 기반으로 LTE 모듈 제어 프로젝트를 신규 생성.

- **UART1** (P0.16 TX / P0.15 RX = RAK_UART2) → LTE 모듈 AT command 통신
- **UART0** (P0.20 TX / P0.19 RX) → Debug console (RS485 경로)
- UART1에 인터럽트 기반 RX 수신 (ring buffer 512B)
- MCP23017 GPA0 → LTE_RST_N 리셋 제어 (LOW 500ms → HIGH → 5초 부팅 대기)
- AT 명령 송수신 함수 구현 (lte_send_at, lte_recv, at_cmd)
- 초기화 시퀀스: ATE0 → ATI → AT+CPIN? → AT+CREG? → AT+CSQ → AT+CGATT? → AT+COPS?
- 15초 간격 CREG/CSQ 모니터링 루프

#### 3.2 LTE_FLOW.md 전체 제어 흐름 문서

RC76xx AT Command Reference (Rev4.0) 및 Product Technical Specification (Rev.12) PDF를 분석하여 작성.

**Phase 1~10**: HW 초기화 → 모듈 리셋 → AT 통신 확인 → 모듈 정보 → SIM 확인 → 네트워크 등록 → 신호/통신사 → PDP Context → 데이터 통신 (TCP/UDP/HTTP) → 주기적 모니터링

각 Phase별 AT 명령어, 파라미터, 응답 예시, 에러 처리 포함.

#### 3.3 SKU 비교 및 한국 밴드 분석

- RC76xx 8개 모델 기능 비교 (LTE Cat, 음성, GNSS)
- GNSS는 전 모델 SKU-dependent (주문 옵션에 따라 GPS 포함/미포함)
- 한국 LTE 밴드 분석: RC7611은 북미 전용(B1,B3 미지원), **RC7620/RC7630이 한국 권장**
- SKU 확인 명령어: ATI, AT!SKU, AT!GNSSCONFIG?

#### 3.4 GNSS (GPS) 제어 흐름 문서

- GNSS 엔진: QUALCOMM iZat Gen8C (GPS L1, Galileo E1, BeiDou B1, GLONASS L1)
- 성능 사양: TTFF Cold 32초, 정확도 < 2m, 추적 감도 -160dBm
- 7단계 제어 흐름: Constellation 설정 → NMEA 설정 → 측위(1회/연속) → 상태/위치/위성 조회 → 종료
- 전체 GNSS AT 명령 상세 (파라미터, 예시)

### 8. 핵심 파악 사항

- **RC76xx UART**: UART1(8-wire, primary), UART2(4-wire, debug only). UART1 지원 baud: 2400~4Mbps
- **Power-up**: POWER_ON_N LOW 200ms~7s 펄스, 부팅 시간 typ 7초 / max 24초
- **RESET_IN_N**: MCP23017 GPA0으로 제어 가능 (Active Low, 500ms 펄스)
- **한국 밴드**: RC7611은 B1,B3 미지원(북미 전용), RC7620/RC7630이 한국 통신사 호환
- **GNSS**: SKU-dependent, AT!CUSTOM="GPSENABLE" 로 활성화 가능, QUALCOMM iZat Gen8C
- **TCP/UDP/HTTP**: 모듈 내장 프로토콜 스택 (+KTCPCFG, +KUDPCFG, +KHTTPCFG)
- **AT 명령 최대 길이**: 513자
