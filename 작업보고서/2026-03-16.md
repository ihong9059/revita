### 4. 발견된 하드웨어 이슈

#### GPA1, GPA4 토글 시 시스템 리셋

**증상**: GPA1 또는 GPA4를 ON하면 즉시 시스템 리셋 발생

**분석**:
- GPA1 = X_EN_P2, GPA4 = Y_EN_P2 (P2 관련 신호)
- 해당 핀에 연결된 하드웨어에서 과전류 또는 단락 추정
- 모터 드라이버 P2 enable 신호로 보이며, 하드웨어 점검 필요

**권장 조치**:
- X_EN_P2, Y_EN_P2 연결 회로 점검
- 해당 신호선의 부하 확인

### 5. 최종 펌웨어 기능

#### UART 명령 제어

| 명령 | P0.04 | P0.05 | GPA2 | GPA3 | 설명 |
|------|-------|-------|------|------|------|
| '0' | OFF | OFF | OFF | OFF | STOP (모두 OFF) |
| '1' | ON | OFF | OFF | ON | Mode 1 |
| '2' | OFF | ON | ON | OFF | Mode 2 |

- 1초마다 카운트 및 현재 상태 출력
- 명령 입력 시 카운트 리셋

### 7. 주요 코드 구조

```c
// UART 명령에 따른 GPIO 제어
if (c == '0') {
    // STOP: 모두 OFF
    gpio_pin_set(gpio0_dev, PIN_04, 0);
    gpio_pin_set(gpio0_dev, PIN_05, 0);
    mcp23017_write_reg(MCP23017_GPIOA, 0x00);
} else if (c == '1') {
    // Mode 1: P0.04 ON, GPA3 ON
    gpio_pin_set(gpio0_dev, PIN_04, 1);
    gpio_pin_set(gpio0_dev, PIN_05, 0);
    mcp23017_write_reg(MCP23017_GPIOA, GPA3_MASK);
} else if (c == '2') {
    // Mode 2: P0.05 ON, GPA2 ON
    gpio_pin_set(gpio0_dev, PIN_04, 0);
    gpio_pin_set(gpio0_dev, PIN_05, 1);
    mcp23017_write_reg(MCP23017_GPIOA, GPA2_MASK);
}

// 1초마다 상태 출력
[0001] Mode 1 | P0.04=1 P0.05=0 | GPA2=0 GPA3=1
```
