### 4. 주요 작업 내용

#### 4.1 LoRa RX 수신 불능 이슈 분석

- **증상**: target2(Gateway)가 TX, target1(kc_firmware)이 RX인데 수신이 안 됨
- **TX/RX 파라미터 비교**: 주파수(922.1MHz), SF9, BW125, CR4/6, AES 키/논스 모두 일치 확인
- **원인 발견**: target2 콘솔에서 `[GW] TX failed: -11` 반복 확인
  - 에러 -11 = EAGAIN: SX1262 라디오가 TX_DONE 인터럽트(DIO1)를 받지 못해 타임아웃
  - 처음 3개 패킷은 성공 후 이후 계속 실패 → 라디오 state machine이 stuck

#### 4.2 lora_gateway 수정 사항

`lora_gateway/src/main.c` TX 루프 수정:

```c
/* 변경 전: TX 실패 시 그냥 로그만 출력 */
if (ret < 0) {
    printk("[GW] TX failed: %d\n", ret);
}

/* 변경 후: TX 실패 시 라디오 재설정, 매 TX 성공 후에도 재설정 */
if (ret < 0) {
    printk("[GW] TX failed: %d, reconfiguring...\n", ret);
    lora_config(lora_dev, &config);  /* 라디오 상태 리셋 */
    k_sleep(K_MSEC(100));
    continue;
}
/* ... TX 성공 로그 ... */
lora_config(lora_dev, &config);  /* 다음 TX 전 재설정 */
```

#### 4.3 target2 플래시 과정

1. 첫 리셋 시 HardFault 상태 발견 (PC=FFFFFFFE)
2. 칩 전체 erase 후 재플래시
3. 부팅 메시지 정상 출력 확인
4. TX 3개 성공 후 -11 에러 재현 확인
5. 코드 수정 후 재빌드 & 재플래시
