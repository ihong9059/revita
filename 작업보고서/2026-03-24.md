### 5. LoRa Star Topology 통신 설계

#### 5.1 토폴로지 구조

```
                    ┌─────────────────┐
                    │   리비타 타워    │
                    │   (Hub/Gateway) │
                    │                 │
                    │  - LTE → 서버   │
                    │  - LoRa Master  │
                    └────────┬────────┘
                             │
            LoRa Star Topology (920.9MHz)
                             │
         ┌───────────────────┼───────────────────┐
         │                   │                   │
         ▼                   ▼                   ▼
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│  리비타 링크 #1  │ │  리비타 링크 #2  │ │  리비타 링크 #N  │
│     (Node)      │ │     (Node)      │ │     (Node)      │
│                 │ │                 │ │                 │
│  - 센서 수집    │ │  - 밸브 제어    │ │  - 유량계       │
│  - 밸브 제어    │ │  - 센서 수집    │ │  - 센서 수집    │
└─────────────────┘ └─────────────────┘ └─────────────────┘
```

#### 5.2 Star Topology 선택 이유

| 항목 | Star Topology | Mesh Topology |
|------|---------------|---------------|
| 복잡도 | 낮음 ✅ | 높음 |
| 전력 소모 | 낮음 ✅ | 높음 (라우팅) |
| 지연 시간 | 짧음 ✅ | 가변적 |
| 라이선스 | 자체 구현 ✅ | MeshCore 라이선스 이슈 |
| 확장성 | 제한적 | 유연함 |
| 노지 환경 | 적합 ✅ (타워 중심) | 불필요한 복잡성 |

#### 5.3 LoRa 통신 사양

| 항목 | 값 | 비고 |
|------|-----|------|
| 주파수 | 920.9 MHz | 한국 ISM 대역 |
| Spreading Factor | SF9 | 거리/속도 균형 |
| Bandwidth | 125 kHz | 표준 설정 |
| Coding Rate | CR 4/6 | 오류 정정 |
| TX Power | 14 dBm | 규정 준수 |
| 암호화 | AES-128-CCM | TinyCrypt 라이브러리 |

#### 5.4 기존 구현 현황 (2026-03-18 완료)

| 프로젝트 | 설명 | 상태 |
|---------|------|------|

| `loraSecure_hub/node` | AES-128-CCM 암호화 추가 | ✅ 테스트 완료 |

#### 5.5 통신 프로토콜 흐름

```
[타워 (Hub)]                    [링크 (Node)]
     │                               │
     │── TX: 명령/요청 ──────────────▶│
     │                               │ 수신 및 처리
```
