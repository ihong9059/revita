### Console 매핑 (★중요 - Tower 와 Link 가 다름)
| 보드 | Zephyr console | TX 핀 | RX 핀 | host device | baud |
|------|----------------|-------|-------|-------------|------|
| Tower | `&uart0` | P0.20 (RAK_UART1_TX) | P0.19 | /dev/ttyUSB1 | 115200 (대부분) / 9600 (RS485 일 때) |
| Link  | `&uart1` | P0.16 (RAK_UART2_TX) | P0.15 (RAK_UART2_RX) | /dev/ttyUSB0 | 115200 |

### 핵심 핀 매핑
**Tower (tower.h)**
- I2C0: P0.13 SDA / P0.14 SCL → MCP23017 @ 0x20
- RS485: uart0 P0.20/P0.19 → TMUX1574 → 트랜시버, DE=P0.17 RE#=P0.21 (직접 GPIO)
- LTE: uart? P0.16/P0.15 (RAK_UART2 → LTE 모듈, 회로도)
- BAT_AIN: P0.05 / AIN3, 분압 1M/220k → ×5.545
- VIB_SENSE: P1.04, BTN: P1.02
- MCP23017 GPA: GPA0=LTE_RST# GPA2=12_14V_EN GPA3=MUX_SEL GPA4=MUX_EN# GPA5=3V3_EN GPA6=5V_EN GPA7=CAM_EN
- MCP23017 GPB: GPB0=SBC_RST# GPB1=BUZZER_EN GPB6=LED_EN

**Link (link.h)**
- RS485: uart0 P0.20/P0.19, DE=P1.04 RE#=P1.03 (직접 GPIO)
- 12V_EN=P0.17 (직접), BUZZER_EN=P0.24 (직접)
- BAT_AIN: P0.31 / AIN7, 분압 1M/1M → ×2
- BTN: P0.05/AIN3, VIB_SENSE: P0.21
- DIO_X=P0.10 / DIO_Y=P0.09 (NFC 핀, GPIO 사용 시 UICR.NFCPINS off 필요)
- X 채널: P0.13(EN_A) P0.14(EN_B) P0.04(EN_P2)
- Y 채널: P0.25(EN_A) P1.01(EN_B) P1.02(EN_P2)
- QSPI: P0.02/03/26/28/29/30 (모듈 내부)

### Tower MCP23017 I2C 진단 (오늘 핵심 발견)
- 기본 board overlay 만으로는 MCP 가 응답 안 함 (rc=-2)
- **수정 필요**: overlay 에 i2c0 핀 명시
  ```
  &i2c0 {
      clock-frequency = <I2C_BITRATE_STANDARD>;  // 100kHz, 400kHz 아님
      pinctrl-0 = <&i2c0_default_revita>;
      pinctrl-1 = <&i2c0_sleep_revita>;
      pinctrl-names = "default", "sleep";
  };
  &pinctrl {
      i2c0_default_revita: i2c0_default_revita {
          group1 {
              psels = <NRF_PSEL(TWIM_SDA, 0, 13)>,
                      <NRF_PSEL(TWIM_SCL, 0, 14)>;
          };
      };
      i2c0_sleep_revita: i2c0_sleep_revita {
          group1 {
              psels = <NRF_PSEL(TWIM_SDA, 0, 13)>,
                      <NRF_PSEL(TWIM_SCL, 0, 14)>;
              low-power-enable;
          };
      };
  };
  ```
- 코드에서 `k_msleep(100)` 안정화 + 0x20-0x27 probe
- buzzer_test/tower 에서 검증됨 → 다른 Tower 프로젝트도 이 패턴 사용

### LoRa 파라미터 (Gateway/Node 동일)
- 922.1 MHz, **SF12**, BW 125 kHz, CR 4/6, 14 dBm, Preamble 8, 32B payload
- SF12 사용 이유: 안테나 약함 보완 (RSSI -46~-65 양호)
- SF9 로 낮춰 비교 테스트는 미수행

### RS485 timing (학습)
- 9600 baud: 1 byte ≈ 1.04 ms
- nRF UARTE `uart_poll_out` 는 ENDTX 까지만 보장 → shift register flush 안 됨
- TX→RX 전환 전 **k_busy_wait(2000)** (2 ms) 필요 — 1 byte 시간 + 여유

### 디버깅 이력 (해결됨)
1. **Link 디버그 UART 핀**: P0.20 (Tower 와 동일 시도) ❌ → P0.16/P0.02 (kc_cert) ❌(RX 동작 안 함) → **P0.16/P0.15 (uart1)** ✅
2. **RS485 wire `\r\n` 깨짐**: `k_busy_wait(2000)` 으로 해결
3. **cutecom 보드레이트 불일치**: 115200 → 9600 (RS485)
4. **Tower MCP I2C**: 핀 명시 + 100kHz 클럭 → 정상 응답

### 미해결 이슈
- **Tower RS485 wire 출력**:
  - 코드는 kc_cert/tower 패턴 + i2c0 핀 명시 + 3V3/5V/12V_EN 모두 HIGH
  - ttyUSB1 디버그 콘솔 출력은 정상
  - RS485 A/B 라인에 신호 미도달
  - **다음 시도**: HW 점검 — 멀티미터로 트랜시버 VDD, MUX 출력, A/B 라인 측정
- **NFC 핀 GPIO 재할당** (Link DIO_X/Y): UICR.NFCPINS off 필요 (CONFIG_NFCT_PINS_AS_GPIOS=y)
