### 3. 주요 작업 내용

#### 3.1 Tower 프로젝트 회로도 분석

- REVITA_TOWER_v1.pdf 회로도에서 MCU_PART, UART_MUX 섹션 분석
- PORT_MAP.md 문서 작성 (MCU 핀 할당, MUX 채널 매핑, RS485 트랜시버 연결)
- PDF 파일을 tower 프로젝트에 복사

#### 3.2 TMUX1574PWR MUX IC 파악

- 초기에 74HC4052 (dual 4:1, S0/S1 2개 셀렉트)로 잘못 분석
- 사용자 확인으로 **TMUX1574PWR** (4ch 2:1 MUX, SEL 1개) 임을 파악
- D1~D4가 MCU 쪽 공통, SEL=LOW → A그룹(RS485), SEL=HIGH → B그룹(SBC)

#### 3.3 MCP23017 I2C 제어 구현

- MCP23017 I2C 주소 0x20 확인 (I2C 버스 스캔으로 검증)
- GPA3 = MUX_SEL (LOW=RS485), GPA4 = MUX_EN# (LOW=Enable)
- 부팅 시 MCP23017 초기화 → MUX 활성화 → RS485 출력 경로 확보
- **타이밍 이슈 해결**: MCP23017 power-on reset 대기 (100ms) 필요

#### 3.4 RS485 Half-Duplex 방향 전환

- DE(P0.17), RE#(P0.21)을 TX 시에만 HIGH로 전환
- 평소에는 RX 모드 (DE=LOW, RE#=LOW)
- TX 전후 안정화 딜레이 추가 (100us 전, 500us 후)
- uart_poll_out/uart_poll_in으로 RS485 송수신 구현

#### 3.5 J-Link 다중 타겟 관리

- target1 (S/N: 1050295470) = Tower 보드
- target2 (S/N: 1050234191) = Link 보드 (kc_firmware)
- `--dev-id` 옵션으로 개별 플래시 가능 확인
- J-Link 연결 불안정 이슈 발생 → 재연결로 해결

## 세션2: LoRa System Loopback 테스트

### 세션2 목표
- Tower↔Link 간 LoRa 양방향 통신 테스트 (loopback)
- `kc_cert/system/` 프로젝트 신규 생성 (tower + link 분리)

### 세션2 작업 결과 요약

| 항목 | 상태 | 비고 |
|------|------|------|
| system 프로젝트 생성 (tower/link) | ✅ 완료 | `kc_cert/system/tower/`, `kc_cert/system/link/` |
| Tower LoRa TX | ✅ 완료 | 3초 간격 "TOWER:N" 송신, 단일 스레드 TX→RX 순차 |
| Link LoRa RX + loopback TX | ✅ 완료 | 수신 후 "LINK:TOWER:N" 접두어 붙여 loopback |
| Tower LoRa RX (loopback 수신) | ✅ 완료 | loopback 수신 후 uart1 + RS485 듀얼 출력 |
| Tower RS485 출력 (MCP23017+MUX) | ✅ 완료 | 기존 tower 코드 재활용 |
| SF9에서 안정 통신 | ❌ 실패 | Tower TX RSSI=-113, SNR=-17 (안테나 문제) |
| SF12로 변경 후 안정 통신 | ✅ 완료 | 패킷 손실 0% 달성 |

### 세션2 주요 작업 내용

#### S2.1 system 프로젝트 구조 설계
- 기존 `kc_cert/tower/` (RS485+MCP23017)와 `kc_cert/kc_firmware/` (LoRa+센서) 참고
- `kc_cert/system/tower/`: LoRa TX → RX 대기, uart1 debug + uart0 RS485 듀얼 출력
- `kc_cert/system/link/`: LoRa RX → debug 출력 → loopback TX

#### S2.2 LoRa TX Config 실패 디버깅
- 최초 RX thread + main TX 구조에서 `lora_config(true)` 실패 (-EIO)
- **원인**: RX thread가 `lora_recv()`에서 blocking 중에 TX config 시도
- **해결**: 단일 스레드 TX→RX 순차 처리로 변경

#### S2.3 Link lora_recv -EIO 디버깅
- Link에서 `lora_recv()` 첫 호출 시 -5 (EIO) 반환
- **원인**: SX1262 초기화 직후 RX 진입 시 radio 미안정
- **해결**: warmup TX 1회 수행 후 RX 진입 → -EIO 해소

#### S2.4 안테나 비대칭 문제 발견
- Tower→Link: RSSI=-113, SNR=-17 (매우 약함)
- Link→Tower: RSSI=-99, SNR=+6 (양호)
- **결론**: Tower 보드 안테나 문제 (TX 출력 약함)
- **임시 해결**: SF9→SF12로 변경하여 감도 ~7.5dB 개선 → 안정 통신

### 세션2 핵심 파악 사항

- **SX1262 warmup**: 초기화 직후 바로 RX 진입하면 -EIO, TX 1회 후 RX 하면 정상
- **LoRa TX/RX 전환**: 단일 디바이스에서 TX↔RX 전환 시 `lora_config()` 재호출 필요
- **Zephyr lora_recv**: 매 호출마다 `Radio.Rx(0)` 실행하므로 재설정 없이 반복 가능
- **west 빌드 주의**: `west build`/`west flash`는 가장 가까운 build 디렉토리를 찾으므로, link 빌드 시 반드시 `cd link/` 후 `-d build` 지정 필요
- **Tower 안테나 비대칭**: TX 약함 (RSSI=-113), 현장 안테나 점검 필요

### 세션3 주요 작업 내용

#### S3.1 tower_port.h 헤더 파일 생성
- REVITA TOWER V1 보드의 전체 포트 할당을 하나의 헤더로 정리
- **MCU Direct GPIO**: I2C(P0.13/14), UART0 RS485(P0.19/20), UART1 debug(P0.15/16), RS485 DE/RE#(P0.17/21), SPI, ADC, LED, 진동센서, 버튼
- **MCP23017 GPA**: LTE_RST#, 12V_EN, MUX_SEL, MUX_EN#, 3V3_EN, 5V_EN, CAM_EN
- **MCP23017 GPB**: SBC_RST#, BUZZER_EN, LED_EN
- 미확인 핀도 주석으로 기록 (USB D+/D-, LTE 제어, P0.26/P1.01)

#### S3.2 main.c 매직넘버 제거
- `#include "tower_port.h"` 추가
- 로컬 `#define REG_IODIRA`, `GPA3_MUX_SEL`, `GPA4_MUX_EN` 제거 → 헤더 참조
- `gpio_pin_set(gpio0_dev, 17, ...)` → `PIN_RS485_DE`
- `gpio_pin_set(gpio0_dev, 21, ...)` → `PIN_RS485_RE_N`
- `REG_IODIRA` → `MCP_REG_IODIRA`, `REG_OLATA` → `MCP_REG_OLATA`
- `GPA4_MUX_EN` → `MCP_GPA4_MUX_EN_N` (Active Low 명시)
