## 1. 현재 상태 분석

### 1.1 아키텍처 개요

```
┌─────────────────────────────────┐     ┌─────────────────────────────────┐
│      Target1 (Central)          │     │      Target2 (Peripheral)       │
│      J-Link: 001050234191       │     │      J-Link: 1050295470         │
├─────────────────────────────────┤     ├─────────────────────────────────┤
│  BLE Central                    │     │  BLE Peripheral                 │
│  - 스캔 → 연결 → 데이터 전송    │◄───►│  - 광고 → 연결 대기 → Echo     │
├─────────────────────────────────┤     ├─────────────────────────────────┤
│  LoRa TX                        │     │  LoRa RX (별도 스레드)          │
│  - 1초마다 카운터 전송          │────►│  - 500ms 타임아웃 수신 → Echo   │
│  - 전송 후 500ms Echo 대기      │◄────│                                 │
└─────────────────────────────────┘     └─────────────────────────────────┘
```

### 1.2 현재 파라미터 설정

| 항목 | Central | Peripheral | 비고 |
|------|---------|------------|------|
| BLE Scan Interval | 60ms (0x0060) | - | BT_GAP_SCAN_FAST_INTERVAL |
| BLE Scan Window | 30ms (0x0030) | - | BT_GAP_SCAN_FAST_WINDOW |
| BLE Adv Interval | - | 100-150ms | BT_GAP_ADV_FAST_INT_MIN/MAX_2 |
| BLE Connection Interval | 100ms | 100ms | min=max=80 (80*1.25ms) |
| BLE Supervision Timeout | 4s | 4s | 400 * 10ms |
| BLE TX Power | +8 dBm | +8 dBm | CONFIG_BT_CTLR_TX_PWR_PLUS_8 |
| LoRa Frequency | 923 MHz | 923 MHz | KR 대역 |
| LoRa SF | 7 | 7 | Spreading Factor |
| LoRa BW | 125 kHz | 125 kHz | Bandwidth |
| LoRa TX Power | 14 dBm | 14 dBm | |
| LoRa RX Timeout | 500ms | 500ms | K_MSEC(500) |

### 1.3 확인된 문제점

| 문제 | 증상 | 추정 원인 |
|------|------|-----------|
| BLE 연결 간헐적 실패 | REVITA_RX 발견 후 연결 실패 | 스캔/광고 타이밍 불일치 |
| BLE 연결 끊김 | 연결 후 간헐적 disconnect | LoRa 처리 중 BLE 이벤트 지연 |
| LoRa Echo 손실 | TX 후 Echo 미수신 | Central TX↔Peripheral RX 타이밍 충돌 |

## 2. 시험 항목

### Phase 1: BLE 단독 안정성 테스트

#### 테스트 1.1: 광고 간격 최적화
**목적**: Peripheral 광고 간격 조정으로 발견율 향상

| 테스트 케이스 | 광고 간격 | 예상 결과 |
|---------------|-----------|-----------|
| A (현재) | 100-150ms | 기준선 |
| B | 50-75ms | 발견율 향상, 전력 증가 |
| C | 20-40ms | 최대 발견율, 전력 최대 |
| D | 150-200ms | 발견율 저하, 전력 절감 |

**측정 항목**:
- 연결 성공률 (100회 시도)
- 평균 연결 시간
- 전류 소모 (옵션)

**코드 변경 위치**: `peripheral/src/main.c` - `adv_param`
```c
// 테스트 B 예시
static struct bt_le_adv_param adv_param = BT_LE_ADV_PARAM_INIT(
    BT_LE_ADV_OPT_CONN,
    BT_GAP_ADV_FAST_INT_MIN_1,  // 30ms
    BT_GAP_ADV_FAST_INT_MAX_1,  // 60ms
    NULL
);
```

#### 테스트 1.2: 스캔 파라미터 최적화
**목적**: Central 스캔 효율 개선

| 테스트 케이스 | Interval | Window | Duty Cycle |
|---------------|----------|--------|------------|
| A (현재) | 60ms | 30ms | 50% |
| B | 30ms | 30ms | 100% (연속) |
| C | 100ms | 50ms | 50% |
| D | 60ms | 60ms | 100% |

**측정 항목**:
- REVITA_RX 발견 시간
- 연결 성공률
- 스캔 중 전류 소모

**코드 변경 위치**: `central/src/main.c` - `start_scan()`

#### 테스트 1.3: 연결 파라미터 최적화
**목적**: 연결 안정성 및 지연 시간 개선

| 테스트 케이스 | Conn Interval | Latency | Supervision TO |
|---------------|---------------|---------|----------------|
| A (현재) | 100ms | 0 | 4s |
| B | 50ms | 0 | 2s |
| C | 30ms | 0 | 1s |
| D | 100ms | 4 | 4s |

**측정 항목**:
- 연결 유지 시간 (10분 테스트)
- 데이터 왕복 지연 (RTT)
- 연결 끊김 횟수

**코드 변경 위치**: `central/src/main.c` - `device_found()`
```c

| 테스트 케이스 | 구현 방식 |
|---------------|-----------|
| A (현재) | Main loop에서 순차 처리 |

| C | BLE Connection Interval에 LoRa 동기화 |
| D | 시분할 스케줄링 (BLE 500ms, LoRa 500ms) |

**구현 방안 B 상세**:

K_THREAD_DEFINE(lora_tx_tid, 2048, lora_tx_thread, NULL, NULL, NULL, 7, 0, 0);

static void lora_tx_thread(void)
{
    while (1) {
        k_sleep(K_MSEC(1500));  // BLE와 겹치지 않게 조정
        lora_send_counter(counter++);
    }
}
```

## 6. 참고 자료

### 6.1 BLE 파라미터 정의

```c
// Advertising Interval (0.625ms 단위)
BT_GAP_ADV_FAST_INT_MIN_1  = 0x0030  // 30ms
BT_GAP_ADV_FAST_INT_MAX_1  = 0x0060  // 60ms
BT_GAP_ADV_FAST_INT_MIN_2  = 0x00A0  // 100ms
BT_GAP_ADV_FAST_INT_MAX_2  = 0x00F0  // 150ms

// Scan Interval/Window (0.625ms 단위)
BT_GAP_SCAN_FAST_INTERVAL  = 0x0060  // 60ms
BT_GAP_SCAN_FAST_WINDOW    = 0x0030  // 30ms

// Connection Interval (1.25ms 단위)
// 80 = 100ms, 40 = 50ms, 24 = 30ms
```

### 6.2 LoRa Air Time 계산

SF7, BW125kHz, 5바이트 페이로드:
- Preamble: 8 symbols = 8 * 1.024ms = 8.2ms
- Header + Payload: ~10 symbols = 10.2ms
- 총 Air Time: ~18.4ms
