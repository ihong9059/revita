### 변경 파일
| 파일 | 변경 | 설명 |
|------|------|------|

### 변경 내용

#### 1. Web UI (`index.html`)

**이전**: LED ON/OFF, BUZZER ON/OFF, STATUS, SENSOR READ (6버튼)

**이후**: Node(Link) 전체 기능을 4개 카드로 구성

| 카드 | 구성 요소 |
|------|-----------|
| **Output Control** | LED ON/OFF, Buzzer ON/OFF |
| **Motor** | X축 CW/STOP/CCW, Y축 CW/STOP/CCW + 상태 표시 |
| **Sensors** | 토양센서 #1 온도/습도, 토양센서 #2 온도/습도, 엽온센서 온도/습도 |
| **Status/Counter** | Button, Vibration (상태 dot), Flow #1/#2 (펄스 카운트), Battery (프로그레스 바 + 전압) |

- 다크 테마 (GitHub 스타일)
- 응답 파싱: SENSOR, INPUT, MOTOR ACK 자동 반영
- Battery 3.0~4.2V 범위 → 프로그레스 바 색상 변화 (빨강/노랑/초록)

#### 2. Tower 펌웨어 (`main.c`)

**추가된 명령**:
| 명령 | 응답 |
|------|------|
| `CMD:MOTOR_X_CW` / `CCW` / `STOP` | `ACK:MOTOR_X_CW` 등 |
| `CMD:MOTOR_Y_CW` / `CCW` / `STOP` | `ACK:MOTOR_Y_CW` 등 |
| `CMD:SENSOR_READ` | `SENSOR:soil1_temp=T,soil1_humi=H,soil2_temp=T,...` |
| `CMD:INPUT_READ` | `INPUT:btn=0,vib=0,flow1=N,flow2=N,bat=V` |

**시뮬레이션 상태 구조체** 추가:
- Output: led, buzzer, motor_x, motor_y
- Input: btn, vib, flow1, flow2, bat
- Sensor: soil1/soil2/leaf (temp, humi)

### 비고
- 현재는 시뮬레이션 값으로 동작 (Tower 내부 구조체)
- 실제 연동 시: Tower → LoRa → Node에서 수집한 데이터로 교체
- 유량센서 2개, 배터리는 이번에 새로 추가된 항목

## LTE 세션 프로토콜 문서 신규 작성

### 변경 파일
| 파일 | 변경 | 설명 |
|------|------|------|
| `protocol/LTE_SESSION_PROTOCOL.md` | 신규 | Tower↔Host LTE 주기적 연결 세션 설계 |

### 문서 구성
| 섹션 | 내용 |
|------|------|
| §1 핵심 개념 | 상시 연결 vs 주기적 연결 차이, 세션 정의 |
| §2 세션 흐름 | 10단계 시퀀스 (CONNECT → SESSION_END) + JSON 포맷 |
| §3 Host 설계 | 명령 큐 패턴, 웹 UI 동작 원칙, 데이터 저장 |
| §4 Tower 설계 | QSPI 링버퍼, 메인루프, 세션 실행 함수 |
| §5 스케줄링 | Normal/Frequent/Power-save/On-demand 모드 + 즉시 연결 조건 |
| §6 에러 처리 | 연결 실패 재시도, 세션 중 끊김, 버퍼 overflow |
| §7 보안 | TLS, 인증, 재전송 방지 |
| §8 단계적 전환 | 시뮬레이션 → 명령 큐 → LTE → 실제 연동 |

### 핵심 설계 결정
- **Tower가 항상 연결 시작** (Host는 수동 대기)
- **명령은 큐에 적재 → 다음 세션에 일괄 전달** (실시간 push 불가)
- **UPLOAD_ACK 수신 후에만 QSPI 레코드 삭제** (데이터 손실 방지)
- **Host가 SCHEDULE 메시지로 다음 연결 주기 조절** (상황 적응형)

## Host Web — LTE 세션 프로토콜 기반 전면 재작성

### 변경 파일
| 파일 | 변경 | 설명 |
|------|------|------|
| `host_test/web/app.py` | 전면 재작성 | 세션 관리, 명령 큐, 텔레메트리 스냅샷 |
| `host_test/web/templates/index.html` | 전면 재작성 | 2패널 레이아웃, 세션 기반 UI |

### app.py 주요 변경

**이전**: 단순 시리얼 송수신 + 로그

**이후**: LTE 세션 프로토콜 시뮬레이션

| 기능 | API | 설명 |
|------|-----|------|
| 세션 실행 | `POST /api/session/start` | 텔레메트리 수신 → 명령 전달 → 기록 |
| 세션 히스토리 | `GET /api/session/list` | 과거 세션 목록 |
| 명령 큐 적재 | `POST /api/cmd/queue` | 명령을 큐에 적재 (즉시 실행 아님) |
| 명령 취소 | `POST /api/cmd/cancel` | 대기 중 명령 취소 |
| 대기 명령 | `GET /api/cmd/pending` | 큐에 쌓인 명령 목록 |
| 텔레메트리 | `GET /api/telemetry` | 마지막 세션 스냅샷 |
| 스케줄 설정 | `GET/POST /api/schedule` | 연결 주기/모드 변경 |
| 직접 전송 | `POST /api/send` | 디버그용 직접 명령 |

### index.html 주요 변경

**이전**: 즉시 실행 버튼 + 로그

**이후**: 2패널 레이아웃

| 영역 | 구성 |
|------|------|
| **Header** | REVITA Host, LTE Session Mode 태그 |
| **Session Banner** | 연결 상태, 마지막 세션 시각, 대기 명령 수, 스케줄, Run Session 버튼 |
| **좌측: Sensors** | 6칸 그리드 + 마지막 수신 시각 표시 |
| **좌측: Status/Counter** | Button, Vibration, Flow ×2, Battery |
| **좌측: Output Control** | LED, Buzzer, Motor X/Y — 큐 적재 방식 |
| **우측: Command Queue** | 대기 명령 목록 + 취소 버튼 |
| **우측: Session History** | 세션 기록 (records, cmds, duration) |
| **우측: Schedule** | 모드 선택 (Normal/Frequent/Power Save) + 주기 설정 |
| **우측: Comm Log** | TX/RX/SYS 로그 + 직접 명령 입력 |
| **우측 하단: Connection** | 시리얼 포트 연결 |

### 핵심 UI 원칙 반영
1. **명령 버튼 → 큐에 적재** ("Queued: MOTOR_X_CW" 토스트 표시)
2. **센서값 = 마지막 세션 스냅샷** (수신 시각 표시)
3. **Run Session 버튼으로 수동 세션 실행** (세션 시 텔레메트리 수신 + 명령 전달)
4. **명령 큐 가시성** (pending 목록 + 취소 가능)
